# Telegram Members Parser

Telegram-бот для парсинга участников групп и каналов с сохранением данных в JSON файл.

## 🚀 Возможности

- ✅ Парсинг участников группы/канала
- ✅ Сохранение данных в JSON файл
- ✅ Загрузка участников в историю аккаунта
- ✅ Удаление дубликатов
- ✅ Поддержка интеграции с личным аккаунтом Telegram
- ✅ Экспорт в CSV
- ✅ Статистика по группам

## 📋 Требования

- Python 3.8+
- Telegram API credentials (API ID и API Hash)
- Telethon библиотека

## 🔧 Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/rerty3358-png/telegram-members-parser.git
cd telegram-members-parser
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` и добавьте ваши данные:
```
API_ID=your_api_id
API_HASH=your_api_hash
PHONE=+1234567890
```

## 📖 Как получить API ID и API Hash

1. Перейдите на https://my.telegram.org/
2. Войдите в свой аккаунт
3. Выберите **API Development Tools**
4. Создайте приложение и получите **API ID** и **API Hash**

## 💻 Использование

```bash
python main.py
```

### Команды бота:

- `/start` — начальное приветствие
- `/parse <group_id>` — парсить участников группы
  - Примеры: `/parse -1001234567890` или `/parse @groupname`
- `/list` — показать список загруженных групп
- `/stats <group_name>` — статистика по группе
- `/export <group_name>` — экспортировать в CSV файл
- `/clear <group_name>` — очистить данные группы
- `/help` — справка

## 🔐 Безопасность

- **Никогда** не коммитьте файл `.env` с реальными данными
- Используйте `.env.example` как шаблон
- Храните API_ID, API_HASH и номер телефона в безопасном месте
- Файлы сессии (*.session) создаются автоматически и содержат токен аутентификации

## ⚠️ Важно

- Этот бот работает с личным аккаунтом Telegram (не с ботом)
- Используйте его ответственно в соответствии с Terms of Service Telegram
- Не парсьте больших групп слишком часто, чтобы избежать блокировки
- При первом запуске потребуется двухфакторная аутентификация

## 🛠️ Решение проблем

### SessionPasswordNeededError
Если вы видите эту ошибку, значит на вашем аккаунте включена двухфакторная аутентификация. Введите пароль при запросе.

### Ошибка при подключении
- Проверьте правильность API_ID и API_HASH
- Убедитесь, что номер телефона указан в формате +XXXXXXXXXXX
- Попробуйте удалить файл `*.session` и запустить бота заново
